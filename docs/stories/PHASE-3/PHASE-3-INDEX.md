## Summary

Phase 3 introduces a qualitative daily self-evaluation system (1-5 rating) layered on top of the existing quantitative hours tracking. The feature is split across 3 sequential stories: the schema and UI foundation, streak recalculation based on rating quality, and a points multiplier that rewards higher-rated days.

**Architectural recommendation:** LOW risk (additive schema changes, no data migration), HIGH ROI (qualitative signal significantly enriches the gamification loop).

## Execution Order

### Story 301 (Foundation — start here)
- Database migration: `avaliacao_diaria` column on `metas_diarias`
- Type extension: `Meta` type gains `avaliacao_diaria?: 1 | 2 | 3 | 4 | 5`
- Backend: `updateMeta()` adds `avaliacao_diaria` to `allowedFields`
- Frontend: `DailyRating.tsx` component (5-star interactive)
- Analytics baseline: `avgDailyRating`, `ratingDistribution` added to `useProgressAnalytics`

### Story 302 (Streak — after 301 Done)
- Backend: `atualizarStreakByRating()` function in `pontos.js`
- Route: PATCH `/api/metas/:id` calls streak recalculation when rating is submitted
- Analytics: `activeDaysCount`, `inactiveDaysCount` added to `useProgressAnalytics`
- Tests: edge cases for streak logic

### Story 303 (Points — after 302 Done)
- Backend: `calcularPontos()` extended with rating multiplier (backwards compatible)
- Backend: `ajustarPontosRating()` function applies delta to `pontos_totais`
- Schema: `rating_multiplier` column on `historico_pontos`
- UI: `DailyRating` shows points preview on star hover
- Analytics: `pointsByRating` breakdown added to `useProgressAnalytics`
- Tests: multiplier math, idempotency
